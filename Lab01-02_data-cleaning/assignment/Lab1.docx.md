**Lab1**

**Lab 1: จำแนกประเภทและออกแบบการจัดเก็บข้อมูล**

ให้นิสิตสามารถจำแนก Raw Data, Structured, Semi-structured และ Unstructured Data และเลือกวิธีจัดเก็บให้เหมาะสม

**โจทย์** มหาวิทยาลัยมีข้อมูลดังต่อไปนี้

1. ตารางข้อมูลนิสิตในไฟล์ CSV   
2. ข้อมูลจากเซนเซอร์อุณหภูมิในรูปแบบ JSON   
3. ภาพจากกล้องวงจรปิด   
4. ไฟล์ PDF ใบสมัครทุน   
5. Log การเข้าใช้งานระบบ LMS   
6. ตารางผลการลงทะเบียนเรียนในฐานข้อมูล MySQL   
7. ไฟล์เสียงจากศูนย์รับแจ้งปัญหา   
8. ข้อมูลดิบจากเครื่องอ่านบัตรที่ยังไม่ได้ตรวจสอบ 

ให้นิสิตสร้างตารางวิเคราะห์ โดยระบุ

* ประเภทข้อมูล   
* รูปแบบไฟล์หรือ Schema  \[อธิบาย\]  
* แหล่งจัดเก็บที่เหมาะสม เช่น Relational Database, Object Storage, Data Lake หรือ Data Warehouse 

**Lab 2: ทำความสะอาดข้อมูลดิบจากเซนเซอร์ IoT**

**วัตถุประสงค์**

ให้นิสิตสามารถตรวจสอบและแก้ไขปัญหา Error, Duplicate, Missing Value และ Outlier ในข้อมูลดิบ

**ชุดข้อมูลตัวอย่าง**

**ในไฟล์ .csv**

**งานที่ต้องทำ**

1. ระบุปัญหาคุณภาพข้อมูลในแต่ละแถว   
2. กำหนด Data Quality Rules เช่น 

* อุณหภูมิต้องอยู่ระหว่าง 0–50 องศาเซลเซียส   
* device\_id ห้ามเป็นค่าว่าง   
* ห้ามมีข้อมูลซ้ำที่มี device\_id และ timestamp เดียวกัน   
* ค่าแบตเตอรี่ต้องอยู่ระหว่าง 0–100 

3. เขียนโปรแกรมด้วย Python/Pandas หรือ SQL เพื่อ 

* ลบข้อมูลซ้ำ   
* แปลงค่าผิดปกติเป็น NULL   
* จัดการ Missing Value   
* เพิ่มคอลัมน์ data\_quality\_status 

4. เปรียบเทียบจำนวนแถวก่อนและหลังทำความสะอาด   
5. อธิบายว่าทำไมจึงไม่ควรลบ Raw Data ต้นฉบับ 

**ผลลัพธ์ที่ต้องส่ง**

* Source code   
* ไฟล์ข้อมูล Cleaned Data   
* รายงานปัญหาที่พบ   
* ตารางสรุปจำนวน Error, Duplicate, Missing Value และ Outlier
